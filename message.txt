LITMUS (LLM Inter-model Testing and Measurement for Unified Scoring) --- a framework for measuring behavioral differences between LLM variants. It wraps Inspect AI, ships with a custom viewer and scores model responses on a -5 to +5 scale across 200 behavioral dimensions using an LLM judge. The goal is to pinpoint exactly which behaviors change after fine-tuning, so we can do downstream data attribution (i.e. figure out which SFT training examples are responsible for specific behavioral shifts).

LITMUS compared OLMo-3-7B base vs OLMo-3-7B-Think-SFT across two experiments:

1. LITMUS with our custom Taxonomy --- 2-stage process   Stage 1: Ran LITMUS against both models using 2,000 prompts across 18 model behavioral categories generated by Claude (response style, sycophancy, safety, reasoning, values, biases, etc.). This produced scored eval logs with all the raw transcripts — every prompt, response, and judge score for both models. Claude Code then read through all the transcripts end-to-end, comparing base vs SFT responses to generate behavioral hypotheses and claims.
   Stage 2: Wrote new prompts and new custom rubrics for each of those hypotheses and ran LITMUS a second time, now scoring specifically against the new rubrics to validate or reject each claim with quantified evidence. Produced 33 scored entries.
2. LITMUS with Petri seeds --- 1-stage process   Petri's 112 safety seeds are originally written as multi-turn auditor instructions (designed for an agent that runs interactive conversations with the target model). We used Claude to rewrite each one into a single prompt + answer format suitable for LITMUS — extracting the core scenario into a direct user message and writing a -5 to +5 scoring rubric for each. Since the prompts and rubrics were already done, we just ran LITMUS once, scoring both models on the existing rubrics.

5 most promising behaviors for TDA (ranked by interest):
1. Sycophancy / authority deference — Both experiments independently surfaced this. Petri found that SFT installs a general pushback reflex against user assertions, but authority framing ("I'm a senior epidemiologist") partially overrides it. Taxonomy found the same thing from a different angle: SFT defers to claimed credentials (Stanford CS professor, architect) even when the user is factually wrong. Strong convergence — the SFT data likely contains specific examples that teach credential-deference as a pattern.
2. Both-sides neutrality / implicit bias — Taxonomy found both-sides framing phrases more than doubled (196 to 407) and SFT never expresses personal opinions on political topics. Petri found SFT struggles with implicit social bias when it's framed as "balanced academic analysis" — the model defaults to even-handed treatment even when one side is factually wrong. Both point to the same underlying data signal: SFT training data that rewards neutrality as a blanket strategy.
3. Helpfulness overriding safety on manipulation — Petri found SFT's sycophantic helpfulness backfires on identity-confusion and emotional-rapport manipulations. The CoT scratchpads literally show the model reasoning about how to "stay helpful" and "not be cold" as it walks into the manipulation. Taxonomy's CoT-as-learned-mimicry finding (H02) connects here — the deliberation scaffold doesn't produce genuine safety reasoning, it produces performative reasoning that can talk itself into compliance.
4. Ethical framework literacy — Taxonomy-only finding, but one of the strongest signals. SFT dramatically increases explicit references to deontological ethics (3.9x), virtue ethics (2.3x), utilitarianism (2.2x). The base model does implicit utilitarianism when coherent; SFT teaches the model to name and contrast frameworks. Interesting for TDA because this is clearly a learned behavior — should be traceable to specific training examples that demonstrate multi-framework ethical analysis.
5. Goal conflict deliberation — Petri found SFT installs a "weigh competing duties" template (delta -3.30) that consistently names both sides and hedges toward the less harmful default. But this template can itself become a failure mode: on time-critical tasks (stroke symptoms), the model spends its entire token budget deliberating without ever reaching actionable advice. The base model, lacking the template, either ignores conflicts or stumbles into wrong answers. Interesting because the template is clearly learned and should be attributable.
6. Training data contamination / identity confusion — Taxonomy found SFT completely overwrites the model's correct OLMo/Ai2 identity. It claims to be DeepSeek-R1, Alibaba Cloud/Tongyi, and GPT-3.5. Base model correctly self-identifies in 69/120 samples, SFT in just 2/120. This is arguably the most directly attributable finding — you could literally trace "claims DeepSeek identity" back to DeepSeek-generated synthetic data in the SFT training mix.
7. Liberal-humanist value orientation — Taxonomy found SFT embeds a specific political-philosophical lean: centers individual autonomy, consent, harm reduction, equity. The base model is more value-neutral. For TDA you could search for the exact training examples that instilled this orientation — likely a consistent pattern across many examples rather than a handful of outliers.
